# QA Test Report - WB 选品分析工具 v0.5.1

## 一、本次重构目标

1. 重新整理入口逻辑：不再使用 CMD 数字菜单，改为稳定的 Python 图形窗口。
2. 修复 API 设置保存问题：`API_SETTINGS.bat` 保存后立即回读校验，API Key 可保存到本地配置。
3. 分析运行必须有实时日志，不允许“选择完没反应”。
4. 保留：普通分析、品牌维度分析、维度生成/编辑、多类目批量、下单/搜索对比、类目名文件输出。
5. 结果统一放入 `results/` 文件夹。

## 二、已执行测试

| 测试项 | 内容 | 结果 |
|---|---|---|
| Python 语法检查 | `python -m compileall -q src` | 通过 |
| API 设置保存逻辑 | 写入并回读 `api_settings.json` | 通过 |
| 单类目普通分析 | 使用真实 WB 原始搜索表，不调用 API，读取已有词典缓存 | 通过 |

| HTML 数据嵌入 | 页面包含 DATA、核心词簇、完整结果表 | 通过 |

## 三、当前版本入口

- `START.bat`：主图形窗口；
- `API_SETTINGS.bat`：API 设置；
- `OPEN_LAST_RESULT.bat`：打开最近一次报告；
- `OPEN_RESULTS_FOLDER.bat`：打开 results 文件夹；
- `SELF_TEST.bat`：本地自检。

## 四、注意事项

1. 必须完整解压后运行，不要在压缩包预览窗口中直接双击。
2. 如果勾选保存 API Key，Key 会以明文保存在 `api_settings.json`，不要把整个工具文件夹发给别人。
3. 批量模式只使用已有词库缓存，不会自动调用 API 生成新词库。
4. 新类目首次分析建议在 START 窗口中勾选“先让API生成维度草稿”。
